# KAVEEP-SIA Safety Matrix

Every module defaults to no action. If evidence is insufficient, the module must emit `UNVERIFIED`, `needs_more_review`, `unknown`, `protected`, or `blocked`.

| Module | Allowed actions | Forbidden actions | Risk level handling | Approval requirement | Audit requirement | Failure behavior | Privacy rule | Output type |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Storage Analyzer | Read metadata, count files/folders, measure sizes, identify candidate patterns, record permission errors | Delete, move, rename, overwrite, compress, change metadata, change permissions, execute cleanup | Flag risky locations; never mark protected locations safe | No approval can authorize real action in v0.1 | Record scan target, timing, summary, warnings, errors | Continue on individual errors; mark failed paths as errors or `UNVERIFIED` | Metadata only; no file content reads; local only | Storage report |
| File Classification Engine | Classify by metadata signals, assign category and risk, identify protected and unknown files | Open private contents, alter files, execute scripts, change metadata | Protected/high-risk for system, source, database, config, environment, license, and unknown sensitive files | No approval can authorize real action in v0.1 | Record classification counts, risk counts, protected items, unknown items, errors | Mark uncertain classifications as `unknown` or `needs_more_review` | Metadata only; avoid sensitive path exposure unless detailed reporting is enabled | Classification report |
| Recommendation Engine | Generate advisory recommendations with reasons and risk levels | Apply recommendations, create action queues, execute cleanup, modify files | Protected and high-risk items go to `doNotTouch` or manual review | Explicit approval required before any future action; none performed here | Record each recommendation, reason, risk, protected item, error | Block unsafe or contradictory recommendation candidates | Metadata-derived reports only; no uploads | Recommendation report |
| Decision & Approval Engine | Validate recommendations, detect conflicts, record explicit decisions | Treat silence as approval, run commands, modify files, create cleanup jobs | Block automatic cleanup for protected, high-risk, unknown-risk, and conflicting items | Explicit user approval required for future-action records | Record decisions, rejections, conflicts, protected items, timestamps | Default undecided items to no action; conflicts become blocked | Store decision records locally; metadata only | Decision report |
| Simulation & Execution Planning Engine | Revalidate approved decisions, estimate impact, create dry-run plans | Execute plans, schedule jobs, delete, move, rename, compress, restore | Exclude changed, stale, protected, risky, unknown, or unverified items | Prior approval required even for planning input; planning remains dry-run | Record planned, excluded, blocked, warning, and error items | Exclude any item that fails revalidation | Metadata only; no content inspection; local only | Simulation report |
| Audit, History & Recovery Engine | Append audit records, generate history summaries, prepare recovery metadata | Delete history, rewrite history, execute recovery, restore files, modify user files | Preserve recorded risk and status; never convert unknown to safe | Approval events are recorded, not created by this module | Append every major event, warning, blocked item, report, and error | Append error record; never erase prior history | Local append-only metadata; no private contents; no upload | Audit report |
